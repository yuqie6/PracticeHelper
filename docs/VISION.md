# VISION - PracticeHelper

## 1. 现在是什么

PracticeHelper 是一个面向后端 / AI Agent 方向求职者的面试训练系统。

它当前的核心能力是：
- 根据用户的真实项目和薄弱环节，由 AI 出题、追问、评分、生成复盘卡
- 持续记录弱点并影响后续训练推荐
- 支持基础知识训练（Go / Redis / Kafka）和项目训练（基于 GitHub 仓库）

但它已经不只是一个问答工具。系统已经具备：
- **项目理解**：导入仓库后自动分析代码，生成结构化项目画像
- **用户画像**：采集求职背景，个性化训练内容
- **弱点记忆**：跨会话追踪薄弱点，带严重度和频率，影响推荐
- **推荐闭环**：基于弱点排序生成训练建议

这四个能力组合在一起，已经构成了"诊断 → 训练 → 记忆 → 推荐"的基础循环。

## 2. 未来不是什么

- 不是通用 AI 求职平台（不做简历投递、JD 聚合、面试匹配）
- 不是多用户 SaaS（保持单用户单进程，不追求规模化）
- 不是算法刷题工具（不做在线判题、代码执行）
- 不是 AI Agent 技术展示（不为了 agent 概念而堆复杂编排）

## 3. 未来要长成什么

一个围绕岗位理解、项目证据、弱点记忆和学习规划的个人求职成长系统。

具体来说，未来的 PracticeHelper 应该能回答求职者这四个问题：

1. **我现在哪里弱？**（弱点诊断 — 已有基础）
2. **这个岗位最看重什么？**（岗位理解 — 待接入）
3. **我的项目经历能证明什么？**（项目证据映射 — 待接入）
4. **我接下来该练什么、学什么？**（学习规划 — 待接入）

面试训练仍然是核心入口和主要交互方式，但训练的目标从"答对题"升级为"针对目标岗位，用项目证据证明能力，补上真实短板"。

## 4. 演化阶段

### 阶段 A：训练体验做稳 ✅ 已完成

目标：让现有训练从"能用"变成"好用"。

- 答题反馈从"评分 + 文字"升级为"结论优先 + 改进建议 + 追问意图"
- 弱点记忆增加时间衰减，区分偶发卡壳和稳定弱项
- 复盘卡从总结式变为行动导向（优先修正项 → 推荐下一轮 → 一键继续）
- 推荐质量与训练历史真正绑定

完成标志：连续用一周，训练建议准确、复盘有行动指引、不想砸掉它。

### 阶段 B：接入岗位视角（当前）

目标：让训练从"泛泛练"变成"针对岗位练"。

- 用户可以维护目标岗位的 JD 文本（手动粘贴，不做爬虫）
- 系统从 JD 中提取关键能力要求
- 训练出题和评分开始参考 JD 要求（"这个岗位要求分布式经验，你的回答没有体现"）
- 弱点维度增加"岗位匹配度"视角

不做：JD 搜索 / 自动爬取 / 多岗位对比。

### 阶段 C：项目证据映射

目标：让用户的项目经历变成可引用的能力证据。

- 项目画像从"描述性摘要"升级为"能力标签 + 证据片段"
- 训练评估时可以引用用户项目代码作为正面/反面证据
- 复盘卡开始出现"你的 XX 项目可以证明这一点，建议这样表达"

不做：自动生成简历 / 简历打分 / 项目重写。

### 阶段 D：学习规划闭环

目标：从"练了就忘"变成"有路径感的成长"。

- 基于弱点历史 + 岗位要求 + 项目证据，生成阶段性学习建议
- 训练记录可视化：能力雷达图、弱点趋势、岗位匹配度变化
- 主动推荐"本周重点"

不做：课程推荐 / 视频整合 / 第三方学习平台对接。

### 关键原则

- **每个阶段都可以独立交付** — 不需要做完阶段 D 才有用，阶段 A 做完就已经比现在好很多
- **后一个阶段依赖前一个阶段的基础** — 不跳阶段，不并行展开
- **训练始终是主交互** — JD、证据、规划都是"让训练更有针对性"的手段，不是独立功能

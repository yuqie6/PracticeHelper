## 1. 这份文档只回答什么

它只回答 3 个问题：

1. 为什么按这个顺序做
2. 哪些阶段已经完成
3. 当前主线应该继续补什么

如果有人开始同时谈很多方向，就回到这份文档收口。

## 2. 推进原则

- 先做纵向闭环，再谈模块摊平。
- 先让它真的对自己有用，再谈“像不像完整产品”。
- 长远蓝图和当前主线要分开写：远图看 NORTH_STAR，当前执行看 PLAN。
- 先把规则和结构做稳，再逐步提高 LLM 自由度。
- 训练是主交易面，其他页面围绕训练组织。
- 不跳阶段，不把后面的愿景伪装成当前待办。
- 已经收口的能力不重复当成“新需求”再做一遍。

## 4. 已完成阶段

### 阶段 A：训练体验做稳

已经完成的关键结果：

- 多轮训练体验比早期版本稳定得多。
- 流式状态、反馈表达和复盘入口已经形成可用闭环。
- 弱项记忆和推荐不再只是展示字段，而是开始影响下一轮。

对应记录：
- [ANSWER_FEEDBACK_UX_V2.md](../records/ANSWER_FEEDBACK_UX_V2.md)

### 阶段 B：接入岗位视角

已经完成的关键结果：

- JD 可以被维护、分析并绑定到训练。
- 出题、评估、追问和复盘都能读取 JD 视角。
- dashboard / session / review 都能回看绑定的岗位上下文。

对应记录：
- [JD_TRAINING_STAGE_B.md](../records/JD_TRAINING_STAGE_B.md)

## 5. 当前所处阶段：阶段 C

当前主线已经不是“让功能更多”，而是让已有闭环更像一个真正可持续使用的训练系统。

阶段 C 的主题固定为：

- 训练更深
- 留存更强
- 检索更准
- 审计更清楚

当前阶段为什么还没结束：

- 第一版检索 / RAG 已接上，但证据绑定和召回质量还没完全收口。
- Prompt 实验已经存在，但对比结果还不够稳定、可解释。
- 推荐与待复习已经形成入口，但“是否真能带动下一轮训练”还要继续验证。

阶段 C 什么时候算收口，不在这里展开细项，具体执行看 [PLAN.md](./PLAN.md)。

### 阶段 C 完成判断

- 当前主线的描述在代码、页面和文档里一致。
- 检索、推荐、复习、prompt 实验都不再停留在“第一版能跑”。
- 继续做 live 使用时，用户能明显感知到训练更深、更连续。

## 6. 明确延后

这些方向不是现在的主线：

- chat-first 首页
- 多 agent 协作
- Workspace 级信息架构重写
- 简历联动、JD 抓取、多岗位对比
- 多用户 / 登录 / 权限
- 通用向量数据库平台化能力

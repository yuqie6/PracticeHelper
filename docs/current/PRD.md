## 1. 要解决的问题

准备后端 / AI Agent 方向面试时，真正难的不是“没有题”，而是：

- 不知道现在该先练什么。
- 基础题只会背结论，一追问就露馅。
- 项目做过，但讲不出技术选型、取舍和个人贡献。
- 每次练完都像重新开始，缺少连续记忆和下一步建议。

所以这个产品的目标不是给更多题，而是让训练更连续、更贴近真实岗位和真实项目。

## 2. 目标用户

当前核心用户是一名准备后端 / AI Agent 方向实习或校招的开发者：

- 熟悉 Go / Python
- 做过若干工程项目
- 需要把“做过”变成“讲得明白、扛得住追问”

## 3. 当前必须成立的能力

| 能力 | 当前口径 |
|:-----|:---------|
| 用户画像 | 采集目标岗位、技术栈、阶段、项目和自感弱项，用于个性化训练 |
| 岗位 JD | 支持维护多份 JD、分析岗位要求、把 JD 绑定到训练轮次 |
| 项目导入 | 导入 Git 仓库，生成项目画像，支持后续项目训练 |
| basics 训练 | 支持 10 个显式 topic 和 `mixed` 自动混合训练 |
| 项目训练 | 围绕真实项目、真实上下文和项目画像追问 |
| 多轮 session | 一轮支持 2-5 轮问答，最后落到 review |
| 复盘与推荐 | 生成复盘卡、优先修正项、推荐下一轮训练 |
| 弱项记忆 | 跨 session 记录弱项、趋势和待复习计划 |

## 4. 当前产品边界

### 当前已成立

- 训练已经不是单轮问答，而是多轮 session。
- 岗位 JD 已经接入训练、复盘和推荐链路。
- 项目训练已经基于真实仓库和项目画像工作。
- 历史页、待复习、弱项趋势和推荐下一轮已经可用。
- Prompt 实验与审计页已经存在，但它是辅助面，不是主入口。

### 当前明确不做

- chat-first 首页 / 全局教练 chat 主入口
- 多用户、登录、权限系统
- 算法判题、代码执行
- 简历联动、JD 自动抓取、多岗位对比
- 全真长时模拟面试
- 通用 repo chat 或通用型知识助手

## 5. 页面口径

当前对外页面是 9 个：

| 页面 | 作用 |
|:-----|:-----|
| `/` | 首页，承接 Diagnose / Today / Recommendation |
| `/profile` | 用户画像维护 |
| `/job-targets` | 岗位 JD 管理、分析与默认 JD |
| `/projects` | 项目导入、项目列表和画像维护 |
| `/train` | 训练配置与开始训练 |
| `/history` | 历史 session 列表、筛选和批量导出 |
| `/sessions/:id` | 训练进行中与过程回看 |
| `/reviews/:id` | 复盘卡与推荐下一轮 |
| `/prompt-experiments` | Prompt 版本、实验对比和审计 |

## 6. 成功标准

当前阶段的产品完成标准不是“页面齐了”，而是下面这些行为成立：

- 用户能从画像 / JD / 项目进入一轮真实训练。
- 训练能连续追问，而不是一次性出题结束。
- 训练结束后有明确复盘和下一步建议。
- 弱项和待复习能影响下一轮推荐。
- 出题、评估、复盘和 prompt 选择具备基本可审计性。

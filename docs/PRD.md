# PRD - PracticeHelper

## 1. 产品一句话

一个面向后端 / Agent 求职方向的面试训练 Agent，帮助用户进行八股、项目表达和模拟面试训练，并根据训练结果持续追踪薄弱点。

## 2. 背景与问题

准备实习或校招时，用户常见问题不是“找不到题”，而是：

- 不知道该优先练什么
- 八股会背，但一追问就露怯
- 项目做过，但讲不顺、讲不硬
- 算法练过，但表达思路容易乱
- 练习没有连续性，弱点不会被持续追踪

现有很多刷题 / 八股工具都偏静态，缺乏：

- 针对真实项目的深挖
- 面向岗位方向的定制训练
- 对薄弱点的长期记忆
- 更像“面试官”的连续追问

## 3. 目标用户

### 核心用户（v0）

- 一名准备找后端 / Agent 方向实习的学生开发者
- 熟悉 Go / Python，做过若干工程项目
- 需要把“做过项目”转化为“能在面试里讲出来”

### 后续潜在用户

- 校招 / 实习求职者
- 有项目但不会表达的开发者
- 想系统训练后端八股与项目面试的人

## 4. 用户目标（Jobs to be done）

用户希望：

- 知道自己该练什么
- 快速进入一轮训练，而不是先读一堆资料
- 在回答后获得针对性反馈
- 让 Agent 记住自己的薄弱点
- 围绕自己的真实项目做训练
- 在投递前进行模拟面试

## 5. 核心能力

### 5.1 用户画像初始化

采集以下信息：

- 目标岗位
- 技术栈
- 目标公司类型
- 当前阶段（实习 / 校招）
- 目标投递时间
- 主项目列表
- 自我感知弱项

### 5.2 八股训练

支持按主题训练：

- Go
- MySQL
- Redis
- Kafka / MQ
- 网络 / OS
- Agent / RAG / Tool Calling
- 其他后续扩展主题

训练形式：

- Agent 提问
- 用户回答
- Agent 评分与点评
- Agent 基于回答继续追问

### 5.3 项目面试训练

围绕真实项目进行：

- 项目介绍
- 技术选型
- 难点与 trade-off
- 架构与数据流
- 失败案例与复盘
- ownership 和实际贡献

### 5.4 训练复盘

每次训练生成复盘卡，包括：

- 总评
- 回答亮点
- 主要问题
- 薄弱点标签
- 建议复习主题
- 下次推荐训练项

### 5.5 薄弱点记忆

系统记录：

- 高频答不好的主题
- 项目表达卡壳点
- 被追问后暴露的问题
- 历史训练轨迹

## 6. MVP 范围

### 必做

- 用户画像初始化
- 八股训练（至少 3 个主题）
- 项目训练（至少 2 个项目）
- 训练复盘卡
- 薄弱点记录

### 可延后

- 算法训练
- 全真模拟面试
- 简历联动
- 岗位 JD 解析
- 个性化学习计划自动编排

## 7. 产品形态

v0 采用 `Web-first` 形态，固定页面包括：

- 首页
- 画像页
- 项目页
- 训练配置页
- 训练过程页
- 复盘页

## 8. 技术方案约束

- 前端固定采用 `Vue 3 + Vite + TypeScript + pnpm`
- Go 服务固定采用 `Gin + SQLite + FTS5`
- Python sidecar 固定采用 `FastAPI + LangGraph + Pydantic`
- 前端风格固定采用 `neo-brutalist`
- Python 代码统一使用 `ruff check` 和 `ruff format`
- Go 代码统一接入 `golangci-lint`
- 前端统一接入格式化与 lint 工具

## 9. 仓库导入与项目训练边界

项目训练使用 GitHub 仓库作为主要输入源。

导入流程固定为：

1. 用户输入 GitHub 仓库 URL
2. 系统拉取仓库到临时目录
3. 扫描文本文件白名单
4. 进行 chunk 切分和结构化提炼
5. 生成项目画像草稿
6. 用户确认或补充
7. 系统允许基于该画像进行项目训练

v0 的“全仓自动理解”定义为：

- 扫描源码和文档文本
- 提炼模块划分、技术点、亮点、风险点、可追问点
- 只服务于项目训练

v0 明确不做：

- 通用型 repo 聊天
- 跨仓知识图谱
- AST 级复杂语义分析

## 10. 非目标

v0 不做：

- 在线代码执行 / 判题平台
- 多用户系统
- 复杂组织协作
- 过于复杂的多 agent 编排
- 移动端原生应用

## 11. 成功标准（v0）

如果满足以下条件，说明 v0 有价值：

- 用户能完成 1 次画像初始化
- 用户能完成 1 次八股训练和 1 次项目训练
- 系统能正确输出复盘卡
- 系统能记录并展示薄弱点
- 用户主观感觉“这玩意儿真的在训练我，而不是在陪聊”

## 12. 工程目标

v0 除了产品可用，还需要满足：

- 单用户本地运行即可
- 不依赖额外基础设施
- SQLite 即可持久化核心数据
- sidecar 输出必须是结构化 JSON
- 前后端和 sidecar 的边界足够清楚，后续能继续演进

## 13. 当前已决策项

- `Web-first`
- `Vue + Vite + pnpm`
- `Gin + SQLite`
- `FastAPI + LangGraph`
- `GitHub 仓库导入`
- `严格 MVP`
- `新野兽派（neo-brutalist）` 作为前端视觉方向
